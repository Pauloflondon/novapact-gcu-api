### Kategorien von AI Governance Tools
1. **Bias detection tools** - Identifizieren menschliche Biases in AI
2. **Automated monitoring tools** - Real-time monitoring, audit trails, compliance reporting
3. **Compliance management tools** - Track regulatory requirements
4. **Explainability tools** - Erklären AI decision-making processes
5. **Model lifecycle management** - Development, deployment, monitoring, retirement
6. **Privacy management** - Data privacy oversight

## NovaPact GCU API Einordnung
Die GCU fällt primär in die Kategorien:
- **Automated monitoring tools** (Status-Maschine, HITL)
- **Compliance management tools** (AI Act ready, AUDIT_PACKET.md)
- **Model lifecycle management** (Governance-Workflow)

**Unique Selling Points der GCU:**
- Fokus auf "Compliance-first" statt allgemeine AI governance
- Lightweight, self-hosted (kein SaaS-Lock-in)
- Audit-Export (JSON, PDF, ZIP) - unveränderlich
- Admin Override mit Begründungspflicht
- Vollständige AI Act-Dokumentation
